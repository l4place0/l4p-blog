## 遇到的问题与解决方案

### 问题 1: `create-astro` 交互式命令卡住

**现象**：`npm create astro@latest . --template minimal` 命令挂起，无法非交互式完成。

**原因**：`create-astro` CLI 需要交互式输入，不支持完全自动化。

**解决**：手动创建 `package.json` / `tsconfig.json` / `astro.config.mjs`，跳过脚手架。

---

### 问题 2: `astro:content` 虚拟模块无法在 Vitest 中导入

**现象**：`import { z } from 'astro:content'` 在测试环境中报错 `Cannot find package 'astro:content'`。

**原因**：`astro:content` 是 Astro 构建时的虚拟模块，Vitest 运行时不存在。

**解决**：将 Zod schema 从 `config.ts` 拆分到独立的 `schemas.ts`，直接 `import { z } from 'zod'`。测试导入 `schemas.ts`，Astro 的 `config.ts` 也从 `schemas.ts` 引用。

---

### 问题 3: `getStaticPaths` 无法访问 frontmatter 变量

**现象**：`CATEGORIES is not defined` 构建错误。

**原因**：Astro 的 `getStaticPaths` 在独立上下文中运行，无法访问 frontmatter 中定义的变量。

**解决**：在 `getStaticPaths` 函数内部重新定义常量，或使用硬编码数组。

---

### 问题 4: Pagefind 在构建时报错 `/_pagefind/pagefind.js` 不存在

**现象**：`import('/_pagefind/pagefind.js')` 在 `astro build` 时被 Vite 解析失败。

**原因**：Pagefind 索引在 `astro build` 之后才生成，构建时文件不存在。

**解决**：将搜索相关的 `<script>` 标签改为 `is:inline`，跳过 Vite 处理，运行时动态加载。

---

### 问题 5: GitHub Actions 报错 "account is locked due to a billing issue"

**现象**：工作流触发后立即失败，提示账号锁定。

**原因**：未完全确认（账号计费页面显示正常，`plan` 字段为 `null`）。

**解决**：放弃 Actions 自动部署，改为本地构建 + 推送 `dist` 分支的传统部署方式。

---

### 问题 6: 工作流触发分支不匹配

**现象**：推送后 Actions 未触发。

**原因**：工作流配置 `branches: [main]`，但默认分支是 `master`。

**解决**：改为 `branches: [main, master]`。

---

### 问题 7: GitHub Pages 只渲染骨架，无样式

**现象**：页面只有 HTML 结构，CSS/JS 资源 404。

**原因**：缺少 `.nojekyll` 文件。GitHub Pages 默认用 Jekyll 处理，会忽略 `_` 开头的目录（如 `_astro`）。

**解决**：在 `dist` 根目录添加 `.nojekyll` 文件并推送。

---

### 问题 8: `astro.config.mjs` 有重复 `base` 字段

**现象**：CSS 路径为 `/_astro/` 而非 `/l4p-blog/_astro/`。

**原因**：配置文件中出现两个 `base` 字段，后者覆盖前者。

**解决**：删除重复的 `base: '/'`，保留 `base: '/l4p-blog'`。

## 部署流程

```bash
# 构建
npm run build

# 生成搜索索引
npx pagefind --site /home/l4p/l4.place/blog/dist

# 推送到 dist 分支
cd dist
git add -A && git commit -m "deploy: <描述>"
git push -f origin dist
```

GitHub Pages 配置：`dist` 分支，`/` 根目录。
